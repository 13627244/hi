小游戏大厅 PRD
一、專案概述
專案名稱：多功能小游戏大厅
一句話描述：一个支持PC和手机双模式的网页小游戏集合，包含11款不同类型的互动游戏
版本：1.0
二、問題與目標
問題陳述
现有小游戏缺乏设备适配，PC和手机用户无法获得最佳体验
部分游戏存在控制问题（如贪吃蛇方向键无响应、切水果鼠标操作失效）
游戏入口分散，缺乏统一的游戏大厅界面
移动端缺少虚拟按键支持，触摸操作体验差
目標
创建统一的游戏大厅，支持11款不同类型的小游戏
实现设备预选择机制，用户首次进入时选择PC或手机模式
为每款游戏提供针对性的控制方案：
PC模式：键盘+鼠标操作
手机模式：触摸屏+虚拟按键
修复所有已知的控制bug，确保游戏流畅运行
提供响应式设计，适配不同屏幕尺寸
三、目標用戶
用戶特徵
年龄：8-45岁
设备：PC用户（Windows/Mac）、手机用户（iOS/Android）
技术水平：普通用户，无需专业游戏经验
使用场景：休闲娱乐、打发碎片时间、朋友间互动
用戶需求
希望在不同设备上都能获得良好的游戏体验
需要直观的操作方式，无需复杂学习成本
希望游戏加载快速，无需安装额外软件
需要清晰的游戏说明和操作指引
希望有多种游戏类型可选择，满足不同兴趣
四、用戶故事
用戶故事1：作為手機用戶，我想要在進入遊戲前選擇手機模式，以便獲得針對觸摸屏優化的操作體驗
用戶故事2：作為PC玩家，我想要使用鍵盤和鼠標控制遊戲，以便獲得精確的操作體驗
用戶故事3：作為休閒玩家，我想要在一個統一的遊戲大廳中選擇不同類型的遊戲，以便快速找到感興趣的內容
用戶故事4：作為貪吃蛇愛好者，我想要使用方向鍵控制蛇的移動，以便流暢地進行遊戲
用戶故事5：作為切水果玩家，我想要在PC上用鼠標拖動切水果，在手機上用手勢切水果，以便獲得最佳的遊戲體驗
五、功能需求與驗收標準
功能1：設備模式選擇
描述：用户首次进入网站时，必须先选择PC模式或手机模式，该选择将影响所有游戏的控制方式
驗收標準：
Given 用户首次访问网站
When 用户看到设备选择界面
Then 用户可以点击"电脑模式"或"手机模式"按钮
And 选择后进入游戏大厅主界面
功能2：遊戲大廳
描述：提供11款游戏的统一入口，每款游戏都有详细说明和操作指引
驗收標準：
Given 用户在游戏大厅界面
When 用户点击任意游戏卡片
Then 右侧显示该游戏的详细信息和操作说明
And 点击"开始游戏"按钮后加载对应游戏
功能3：雙模式控制適配
描述：每款游戏根据设备模式提供不同的控制方案
驗收標準：
Given 用户选择了PC模式
When 用户玩贪吃蛇游戏
Then 可以使用方向键控制蛇的移动
Given 用户选择了手机模式
When 用户玩贪吃蛇游戏
Then 屏幕下方显示虚拟方向键，点击可控制蛇的移动
功能4：切水果遊戲修復
描述：修复切水果游戏的控制问题，PC模式支持鼠标拖动，手机模式支持手势识别
驗收標準：
Given 用户在PC模式下玩切水果
When 用户在画面上拖动鼠标
Then 水果被成功切开并计分
Given 用户在手机模式下玩切水果
When 用户在摄像头前做划过手势
Then 水果被成功切开并计分
功能5：貪吃蛇遊戲修復
描述：修复贪吃蛇方向键无响应的问题
驗收標準：
Given 用户在PC模式下玩贪吃蛇
When 用户按方向键
Then 蛇立即响应方向变化并移动
六、技術約束
必須遵守
使用纯HTML/CSS/JavaScript实现，无需外部依赖（除MediaPipe用于手势识别）
所有游戏必须在单个HTML文件中实现
代码必须兼容现代浏览器（Chrome, Firefox, Safari, Edge）
兼容性要求
PC端：Windows/Mac最新版主流浏览器
移动端：iOS 12+ / Android 8+ 的 Chrome 和 Safari
屏幕尺寸：适配从320px到1920px的宽度
不要做
不要使用框架（React/Vue等）
不要使用复杂的构建工具
不要依赖服务器端渲染
不要使用WebGL以外的3D技术
