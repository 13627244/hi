# 小游戏大厅 PRD

## 一、專案概述
- 專案名稱：小游戏大厅
- 一句話描述：一个集成了11款不同类型小游戏的网页应用，涵盖数字猜测、键盘操作、摄像头交互等多种互动体验
- 版本：1.0

## 二、問題與目標
### 問題陳述
用户想要在一个平台上体验多种不同类型的小游戏，但现有的网页游戏通常分散在不同的网站中，体验不一致，且缺乏统一的界面和操作逻辑。用户需要频繁切换网站，注册账户，面对不同的广告和界面设计，这降低了游戏体验的连贯性和便捷性。

### 目標
1. 创建一个统一的游戏大厅，集成多种类型的小游戏
2. 提供一致的UI/UX设计，降低用户学习成本
3. 支持多种交互方式：键盘、鼠标、摄像头、手势识别
4. 实现游戏数据的本地化存储（如最高分记录）
5. 确保所有游戏在单页面中运行，无需页面刷新

## 三、目標用戶
### 用戶特徵
1. **休闲游戏玩家**：寻找轻松、短时间的游戏体验
2. **办公族/学生**：需要短暂休息放松的用户
3. **多人互动场景**：朋友聚会时可以进行双人对战游戏
4. **技术爱好者**：对摄像头交互、手势识别等技术感兴趣的用户
5. **儿童/青少年**：适合培养反应能力和协调能力的游戏

### 用戶需求
1. **便捷性**：快速开始游戏，无需复杂注册
2. **多样性**：不同类型的游戏满足不同心情和需求
3. **互动性**：支持单人、多人、摄像头等不同互动方式
4. **进度保存**：游戏成绩和记录的本地保存
5. **响应式设计**：在不同设备上都能正常体验

## 四、用戶故事
- 用戶故事1：作为办公室工作人员，我想要在午休时间玩一个简单的小游戏放松，以便快速恢复精力，而不会被复杂的游戏机制困扰
- 用戶故事2：作为家长，我想要为孩子找一个既有趣又能锻炼反应能力和手眼协调的游戏集合，以便在娱乐中培养孩子的能力
- 用戶故事3：作为科技爱好者，我想要体验基于摄像头和手势识别的游戏，以便了解前沿的Web技术应用
- 用戶故事4：作为朋友聚会的主持者，我想要找一个支持多人对战的游戏，以便让朋友们一起互动娱乐
- 用戶故事5：作为游戏开发者，我想要参考不同游戏类型的实现方式，以便学习网页游戏开发技术

## 五、功能需求與驗收標準
### 功能1：游戏大厅界面
- 描述：统一的游戏选择界面，显示所有可用游戏，支持游戏切换和描述查看
- 驗收標準：
  - Given 用户打开小游戏大厅页面
  - When 页面加载完成
  - Then 应该显示游戏列表和当前选中游戏的详细信息
  - And 游戏列表应该包含所有11款游戏
  - And 点击不同的游戏按钮应该切换游戏详情

### 功能2：数字猜猜猜游戏
- 描述：猜数字游戏，用户猜测1-20之间的数字，根据提示调整猜测
- 驗收標準：
  - Given 用户选择数字猜猜猜游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示分数面板和输入框
  - When 用户输入数字并点击猜测
  - Then 应该根据猜测结果给出提示（太大/太小/正确）
  - And 分数应该相应变化
  - And 最高分应该被保存到本地存储

### 功能3：经典贪吃蛇游戏
- 描述：键盘控制的贪吃蛇游戏，控制蛇移动吃食物，避免碰撞
- 驗收標準：
  - Given 用户选择贪吃蛇游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示游戏画布和分数
  - When 用户按下方向键
  - Then 蛇应该向相应方向移动
  - When 蛇吃到食物
  - Then 蛇应该变长，分数增加
  - When 蛇撞到墙或自身
  - Then 游戏应该结束

### 功能4：摄像头表情挑战
- 描述：使用摄像头拍摄夸张表情的游戏
- 驗收標準：
  - Given 用户选择表情挑战游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示摄像头控件
  - When 用户点击打开摄像头
  - Then 应该请求摄像头权限并显示视频流
  - When 用户点击开始倒计时
  - Then 应该开始3秒倒计时
  - When 用户点击截图
  - Then 应该捕获当前视频帧并显示为图片

### 功能5：摄像头切水果
- 描述：使用手势识别在空中切水果的游戏
- 驗收標準：
  - Given 用户选择切水果游戏并点击开始
  - When 游戏界面加载
  - Then 应该自动尝试打开摄像头
  - And 应该加载MediaPipe手势识别库
  - When 摄像头成功打开
  - Then 应该显示摄像头画面和水果元素
  - When 用户用手在摄像头前划过水果
  - Then 水果应该被切碎，分数增加

### 功能6：左右手反应对战
- 描述：双人对战游戏，根据提示快速按键得分
- 驗收標準：
  - Given 用户选择左右手对战游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示左右玩家分数和操作说明
  - When 用户按下空格键
  - Then 应该开始新的一轮
  - When 屏幕显示按键指令
  - Then 先按对对应按键的玩家应该得分

### 功能7：反应力点击游戏
- 描述：测试反应速度的点击游戏
- 驗收標準：
  - Given 用户选择反应力点击游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示游戏区域和开始按钮
  - When 用户点击开始按钮
  - Then 应该在随机延迟后显示绿色方块
  - When 用户点击绿色方块
  - Then 应该记录反应时间并更新最高纪录

### 功能8：节奏下落块游戏
- 描述：音乐节奏类游戏，按下对应按键击打下落方块
- 驗收標準：
  - Given 用户选择节奏下落块游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示游戏画布和分数
  - And 应该开始生成下落方块
  - When 方块到达判定线附近
  - And 用户按下对应按键（F/G/H/J）
  - Then 如果时机正确，方块应该消失，分数增加，连击数增加

### 功能9：字母雨打字游戏
- 描述：打字练习游戏，输入下落的字母消除它们
- 驗收標準：
  - Given 用户选择字母雨打字游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示游戏画布和命中/失误统计
  - And 应该开始生成下落的字母
  - When 用户按下键盘上的字母键
  - Then 如果与某个下落字母匹配，该字母应该消失，命中数增加
  - When 字母落到画面底部
  - Then 失误数应该增加

### 功能10：WASD躲避游戏
- 描述：躲避类游戏，控制方块躲避下落的障碍
- 驗收標準：
  - Given 用户选择WASD躲避游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示游戏画布和生存时间
  - And 玩家方块应该出现在画面底部
  - When 用户按下WASD键
  - Then 玩家方块应该相应移动
  - When 玩家方块碰到障碍物
  - Then 游戏应该结束，显示重新开始提示

### 功能11：双手迷宫游戏
- 描述：协调性训练游戏，同时控制两个方块走出迷宫
- 驗收標準：
  - Given 用户选择双手迷宫游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示迷宫地图和两个方块
  - When 左手玩家按下WASD键
  - Then 蓝色方块应该相应移动
  - When 右手玩家按下方向键
  - Then 橙色方块应该相应移动
  - When 两个方块都到达各自终点
  - Then 应该显示完成提示

### 功能12：按键顺序记忆游戏
- 描述：记忆力训练游戏，记住并重复按键序列
- 驗收標準：
  - Given 用户选择按键记忆游戏并点击开始
  - When 游戏界面加载
  - Then 应该显示当前回合和操作说明
  - When 用户按下空格键
  - Then 应该生成并显示按键序列
  - When 序列显示完毕
  - Then 用户应该按相同顺序输入按键
  - When 用户输入正确
  - Then 应该进入下一轮（序列更长）

## 六、技術約束
### 必須遵守
1. 纯前端实现，无需后端服务器
2. 使用原生HTML/CSS/JavaScript，不依赖大型框架
3. 支持现代浏览器（Chrome, Firefox, Safari, Edge）
4. 响应式设计，适配桌面和移动设备
5. 使用本地存储保存游戏数据

### 兼容性要求
1. 摄像头游戏需要用户授予权限
2. 手势识别功能需要MediaPipe库支持
3. 键盘游戏需要完整的键盘支持
4. 移动设备上需要虚拟键盘支持

### 不要做
1. 不要收集用户隐私数据
2. 不要添加 intrusive 广告
3. 不要依赖需要付费的第三方服务
4. 不要使用过时或废弃的Web API
